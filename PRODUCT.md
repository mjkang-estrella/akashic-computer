# Product

## Register

product

## Users

Akashic Computer primarily serves people who are curious about running open-weight models locally but do not yet understand model families, parameter sizes, variants, quantization formats, runtimes, or hardware requirements. Experienced users should still find it useful as a fast, trustworthy discovery and comparison surface.

The primary workflow is to discover a model family, understand the available releases and variants, inspect their artifacts, and then determine which options can run within a generic VRAM budget.

## Product Purpose

Akashic Computer is a discovery atlas for open-weight models. It organizes the ecosystem into a legible hierarchy: family, release, size, variant, and artifact. It presents benchmark evidence, provenance, runtime compatibility, and memory requirements without making subjective recommendations.

Success means a newcomer can understand what options exist and which ones are runnable, while an experienced user can scan and compare the same information without losing detail.

## Brand Personality

Scholarly, calm, and exact. The interface should feel like a well-maintained technical catalog: neutral in its conclusions, confident in its structure, and restrained in its presentation.

## Anti-references

Do not make Akashic Computer feel like a marketing site, affiliate buying guide, generic SaaS dashboard, or promotional AI leaderboard. Avoid universal "best model" claims, recommendation badges, ornamental card grids, decorative gradients, and hardware-first framing that hides the broader model ecosystem.

## Design Principles

1. **Discovery before compatibility.** Show the full model landscape first. Hardware information explains constraints but never determines which models deserve attention.
2. **Explain the hierarchy.** Preserve the path from family to release to size to variant to artifact so newcomers can build a correct mental model.
3. **Evidence without verdicts.** Present benchmarks, provenance, runtime support, and quantization deltas. Do not collapse context-dependent tradeoffs into a single recommendation.
4. **Progressive technical depth.** Make the first scan understandable while keeping expert details available in-place or through expansion.
5. **Trust is visible.** Distinguish official, vendor, community, measured, inferred, and unverified information wherever it appears.

## Accessibility & Inclusion

Target WCAG 2.2 AA. Controls must be keyboard accessible, focus-visible, and usable without relying on color alone. Dense technical content may use compact typography, but essential explanatory text and interactive targets must remain readable and operable.
